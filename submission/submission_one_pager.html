ResQAI

Hyper-Local Generative Disaster Response Coordinator
Problem Statement
In the immediate aftermath of natural disasters (earthquakes, floods, wildfires), conventional communication often fails. Emergency responders are overwhelmed by fragmented data—distress calls from social media, satellite imagery, and IoT sensor streams—leading to delayed response times for critical rescues. There is no unified system that can instantly synthesize this multimodal "noise" into actionable, hyper-local rescue plans for on-ground teams.

Motivation

Disaster response is a race against time. The inspiration comes from the "Golden Hour" principle—survival rates drop drastically after the first 60 minutes. Current systems are reactive and siloed. ResQAI is designed to be a proactive, intelligent agent that actively coordinates the response, acting as a "digital dispatcher" that reasons through chaotic information to save lives.

Application

  • Use Case: A centralized command center dashboard for disaster relief agencies (e.g., NDRF, Red Cross) and a mobile companion app for field units.
  • Target Users: Incident Commanders, Search and Rescue (SAR) teams, and volunteers.
  • Deployment: High-risk zones (e.g., seismic belts, flood-prone deltas) and smart cities integrated with emergency grids.

Proposed Method

ResQAI uses a Retrieval-Augmented Generation (RAG) pipeline powered by a Large Language Model (e.g., Gemini Pro or GPT-4).

  1. Multi-Modal Ingestion: Aggregates text (tweets, SMS), Audio (emergency calls), and Vision (drone/satellite assessment).
  2. Graph-RAG Knowledge Base: Constructs a dynamic Knowledge Graph linking people, hazards, and resources (e.g., "Person trapped" -> "Building B collapsed" -> "Road C blocked").
  3. Generative Planning Agent: Queries this graph to generate optimal route plans and resource allocation strategies.
  4. Feedback Loop: integrates real-time field updates to refine the graph instantly.

Datasets & Data Sources

  • xView2 / CrisisMMD: For damage assessment model validation.
  • Twitter Earthquake Dataset / NIST Trec-IS: For urgency classification training.
  • OpenStreetMap (OSM): For geospatial grounding.
  • Simulated Emergency Calls: Synthetic privacy-safe data for testing.

Experiments & Validation

We will simulate a "Flood Scenario" in a digital twin city environment. Key metrics include:

  • Response Latency: Time from data ingestion to actionable plan generation.
  • Precision/Recall: Accuracy of identifying high-priority rescue targets.
  • Hallucination Rate: Ensuring the AI uses only verified map/resource data.

Novelty & Scope to Scale

Unlike standard dashboards that visualize data, ResQAI coordinates logistics. The use of Graph-RAG for disaster context preserves spatial and relational information often lost in vector-only stores.

Scope: Scalable to Smart Cities for traffic management or large-scale event crowd control. Can operate offline (Edge AI) if cloud connectivity is lost.
